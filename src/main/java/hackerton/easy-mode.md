Easy 문제 유형 정리 (정확도 매우 높음)
1) 문자열(String) 처리 — 가장 압도적으로 많음
자료구조/알고리즘 필요 없음
반복문 + 조건문으로 해결
대표 유형:
부분 문자열 포함 여부(contains)
공백 기준 split
문자열 뒤집기(reverse)
문자 빈도수 세기 (Map or 배열)
두 문자열 공통 문자 여부 (너가 푼 twoStrings)
팔린드롬 판별
예:
“두 문자열이 공통 문자를 갖는가?”
“문자열에서 모음 수 세기”
“단어 빈도 세기”
2) 해시(집합/맵) 기반 문제(Set/HashMap)
HashSet/HashMap만 알면 풀림
시간복잡도 깊은 고민 필요 없음
대표 유형:
중복 문자가 있는지
배열에서 중복 제거
공통 원소 존재 여부
첫 번째 중복 원소 찾기
빈도수 기반 문제
예:
“A 학생 명단과 B 학생 명단이 겹치는지?”
“중복 숫자 있는지?”
3) 배열(Array) 기본 인덱스 다루기
합 구하기(sum)
최소값/최댓값
특정 조건 카운트
배열 비교
4) 정렬(sort) 기반 간단 문제
   Easy에서 정렬이 쓰이면 로직은 거의 쉬움.
   예:
K번째 작은 수 찾기 (정렬 + 인덱스)
두 배열을 정렬해서 비교
정렬 후 최소 차이 구하기
Easy에서 정렬은 붙여넣기 수준이 대부분.
5) 파싱(parsing)
특히 문자열 파싱이 easy에 자주 출제됨.
대표 유형:
CSV 파싱
정수 추출
공백/쉼표 분리
날짜/시간 단순 형식 변환
6) 단순 카운팅(counting)
조건이 조금 있을 뿐 본질은 "세기"
대문자/소문자 수 세기
양수/음수/0 개수 세기
특정 조건 만족 개수 세기
7) 누적합(prefix sum)의 초간단 버전
Easy에서 prefix sum은 거의
“부분 합 구하기"
복잡한 슬라이딩윈도우는 Medium부터 등장.
8) 구현(Implementation) — 아주 기초적인 if/for 문제들
간단한 시뮬레이션
문자열 반복 출력
배열 조건 검사
간단한 stack-like 문제 (push/pop 없고 단순 조건)
Easy 문제의 핵심 특징
정답 길이가 짧음
(보통 5~20줄)
특정 자료구조 “하나”만 쓰면 풀림
Set 하나
Map 하나
배열 한 번 순회
정렬 한 번
슬라이딩윈도우 없음
그래프 없음
DP 없음
Stack/Queue도 거의 안 나옴
